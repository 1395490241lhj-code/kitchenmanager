# MVP 核心流程回归验收清单

每次改动后照着这份清单快速手动过一遍核心路径。建议环境：浏览器 390×844 移动视口（DevTools 设备模拟）+ 本地 `npm start`（http://localhost:3000）。

> 重置到「新用户」状态：DevTools → Application → Local Storage → 清空本站所有 `km_*` key，刷新页面。
> 测完想清掉演示数据：同样清空 Local Storage 即可，不影响代码。

---

## 1. 新用户空库存（Onboarding）

**前置状态**：Local Storage 已清空（无任何 `km_*` key）。

**操作步骤**
1. 打开首页 `#today`。
2. 依次确认四个入口都在：✍️ 手动记食材 / 🧾 拍小票识别 / 🍳 试用示例厨房 / 📖 先逛菜谱。
3. 点「试用示例厨房」。
4. 清空 Local Storage 后重进，点「手动记食材」。

**预期结果**
- 空库存时首页显示引导卡，不显示推荐面板。
- 「试用示例厨房」后首页变成正常 dashboard（有推荐、有统计）。
- 「手动记食材」能打开录入入口；「先逛菜谱」跳到菜谱页。

**常见异常**
- 清空 storage 后仍显示旧数据 → 多半是 SW 缓存了旧 JS，先看流程 10。
- 引导页出现「明天备菜」或推荐面板 → 空库存分支被破坏。

---

## 2. 手动记食材（中文数量解析）

**前置状态**：任意（建议空库存起步）。

**操作步骤**
1. 食材页「随手记几样食材」（或首页「记食材」弹窗的文本批量记）输入四行：
   ```
   鸡肉一份
   鸡蛋三个
   西红柿3个
   牛肉半斤
   ```
2. 点「加入厨房」。
3. 到食材页核对卡片。

**预期结果**
- 库存出现 4 张卡：鸡肉(1 份)、鸡蛋(3 个)、番茄(3 个，西红柿归一)、牛肉(0.5 斤)。
- **不能**出现名为「鸡肉一份」「牛肉半斤」的食材。
- 行内提示「已加入 4 样食材」。

**常见异常**
- 「鸡肉一份」整串成了名字 → 中文数量解析（food-input-parser）回归。
- 「二荆条 / 十三香 / 五花肉」被拆出数量 → 防误解析回归（可顺手输一行验证）。

---

## 3. 首页默认推荐

**前置状态**：库存有 3 样以上常见食材（流程 2 之后即可）。

**操作步骤**
1. 回首页，看顶部主状态文案。
2. 切到「✨ 推荐」tab。
3. 在推荐卡上点「做这道」。

**预期结果**
- 顶部显示「今晚可以做 X 道菜」类主文案（X ≥ 1）。
- 推荐 tab 有本地推荐卡（菜名 + 理由 + 匹配标签），可「换一道」。
- 「做这道」后按钮短暂变「已加入今天」，计划 tab 能看到该菜。

**常见异常**
- 有库存但 0 推荐 → 库存匹配或推荐排序回归。
- 推荐卡出现纯调料菜（如「调味汁」）→ 核心食材分类口径回归。

---

## 4. 指定食材推荐（想用这些食材？）

**前置状态**：流程 3 的库存。

**操作步骤**：推荐 tab 的「想用这些食材？」输入框依次试：
1. `番茄 鸡蛋` → 推荐
2. `蘑菇 鸡肉` → 推荐
3. `绿叶菜 鸡蛋` → 推荐
4. `牛肉 土豆 盐 高汤` → 推荐

**预期结果**
- ① 命中番茄炒蛋类（西红柿=番茄 同义归一）。
- ② 命中香菇/菌菇 + 鸡肉类菜（类别展开）。
- ③ 命中空心菜/青菜/菠菜等绿叶菜 + 鸡蛋类菜。
- ④ 提示行只显示「牛肉、土豆」——盐和高汤被过滤，不参与匹配。
- 每组结果 ≤ 6 道；摘要注明「本地菜谱匹配结果，不调用 AI」。

**常见异常**
- 字面匹配不到就空结果 → 类别/别名展开（ingredient-intent / 食材族）回归。
- 盐、高汤出现在提示或命中里 → 调料过滤回归。

---

## 5. AI 创意做法（需配置 API Key；无 Key 只验证降级）

**前置状态**：流程 4 界面；设置页已配置可用 AI。

**操作步骤**
1. 输入 `芦笋 蘑菇 鸡肉` → 推荐。
2. 本地结果下点「让 AI 想一个做法」。
3. 草稿出现后点「换一种做法」。
4. （无 Key 时）只做第 2 步。

**预期结果**
- 按钮变「正在想... / 正在换个方向...」，期间本地结果不消失。
- 草稿卡明确标「AI 草稿 · 确认后再保存」，食材 pill **不含**盐/水/葱姜蒜/高汤。
- **不自动保存**：不点「保存草稿」就刷新，菜谱库里不应多出这道菜。
- 「换一种做法」应尽量换形态（焖饭/烩饭/烤盘…），而不是只把鸡肉片换成鸡肉丁。
- 无 Key：友好报错「AI 暂不可用…」，本地推荐保持可用。

**常见异常**
- AI 失败把本地结果也清掉 → 状态管理回归。
- 草稿 ingredients 出现调料 → `filterAiDraftCoreIngredients` 回归。

---

## 6. 加入计划 + 备菜提醒

**前置状态**：库存里有一块 `isFrozen: true` 的肉（食材页编辑模式点 🧊 可设）。

**操作步骤**
1. 推荐卡「做这道」加入今天；菜谱详情页「计划明天」加一道含冷冻肉/木耳/腌制做法的菜。
2. 计划 tab 切「只看今天 / 未来 3 天」。
3. 观察计划卡顶部与明天行。

**预期结果**
- 今天/明天的菜分别出现在对应范围；切换不丢数据。
- 明天的菜触发顶部红色小字（如「今晚记得解冻：牛肉」或「今晚需要提前准备：…」）。
- 「未来 3 天」下明天行带小标签（🧊 解冻 / 💧 泡发 / 🧂 腌制）；今天的行不带。
- 「只看今天」时提醒旁有「查看明天计划」可跳转。

**常见异常**
- 普通素菜也触发提醒 → prep-planner 规则回归。
- 后天的菜也提醒 → 只应针对明天。

---

## 7. 买菜清单闭环

**前置状态**：任意；可从菜谱详情「缺少食材加入买菜」造几条。

**操作步骤**
1. 菜谱详情把缺少食材加入买菜；到买菜页确认。
2. 未买项点 ✎：改数量 500、单位 g、备注随意 → 保存。
3. 点整行标记已买 → 在「最近完成」里点「记进厨房」→ 确认。
4. 顶部提示卡「全部记进厨房」（多条已买时）逐条确认。
5. 底部「清除已买」。

**预期结果**
- 行内编辑原地展开（无弹窗），保存后 meta 即显「500 g · 备注」；备注默认值=来源文案（如「菜谱缺货：XX」）可直接改。
- 已买项进「最近完成」不立即消失；记进厨房后显示「已记进厨房」，**不再出现在未买列表**，食材页能看到该食材。
- 清除已买后列表只剩未买项；全空时显示空状态卡。

**常见异常**
- 点 ✎ 把行标成了已买 → stopPropagation 回归。
- 记进厨房后食材页没有 → mergeInventoryEntry 链路回归。

---

## 8. 做完菜（扣减口径）

**前置状态**：库存有鸡腿（无鸡肉）；计划里有一道用「鸡肉」的菜 + 一道含汤/盐的菜。

**操作步骤**
1. 计划行点「🍳 做好了」。
2. 校准弹窗里核对出现的食材清单，确认。
3. 看完成反馈；如有「加入买菜」按钮点一下。

**预期结果**
- 校准清单**只列核心食材**：不出现水、汤、盐、生抽、葱姜蒜、水淀粉。
- 库存只有鸡腿时，「鸡肉」需求能匹配到鸡腿参与扣减（食材族匹配）。
- 确认后库存对应减少/降档；完成反馈可把用完/缺失食材一键加入买菜（只加核心食材）。
- 菜谱页该菜出现「最近做过」状态，推荐对其降权。

**常见异常**
- 弹窗里出现「汤 ×1」「盐」→ computeCookDeductions 分类口径回归。
- 凭空给从未有过的食材建了 0 库存卡 → 幽灵防护回归。

## 9. 主题与移动端

**前置状态**：任意有数据状态。

**操作步骤**
1. 设置页「外观主题」切深色 → 逛首页/食材/买菜/菜谱/详情。
2. 切回浅色，重点看首页空态、按钮、计划卡。
3. 390px 视口下逐页横向滚动检查；滚到底部看浮动导航。

**预期结果**
- 深浅两套下文字均可读，玻璃卡有层次（深色不是死黑/纯灰板）。
- 深→浅切换后**无深色残留**（尤其空态条、chip、输入框底色）。
- 任何页面无横向滚动条；底部导航不遮挡主要操作按钮（内容可滚动到导航之上）。

**常见异常**
- 切回浅色后某块仍是深底 → 写死颜色未走 token / `html[data-theme]` 覆盖缺失。
- 低位按钮被导航盖住且滚不上来 → `#app` 底部留白（112px）被改动。

---

## 10. PWA 缓存

**前置状态**：本次改动涉及 JS / CSS / data。

**操作步骤**
1. 确认 `sw.v18.js` 顶部 `CACHE_NAME` 已 +1（仅 docs/测试改动可不 bump）。
2. 部署/刷新后 DevTools → Application → Cache Storage：只剩新版本 `km-vNN`，旧版本被清理。
3. 强刷一次，确认页面行为是新代码（找一处本次改动的可见变化验证）。

**预期结果**
- Cache Storage 只有当前版本；JS/CSS/data JSON 走 networkFirst 能拿到最新。

**常见异常**
- 改了 JS 但忘 bump → 用户拿旧逻辑，临时解法：DevTools Unregister SW + 清缓存。
